# 달리기 지수 알고리즘

달리기 지수(Running Index)는 0~100점으로, 점수가 높을수록 달리기에 적합한 환경입니다.

## 점수 계산

```
점수 = 100 - (대기질 페널티 합산 + 기상 페널티 합산)
```

기상 데이터(`KMA_API_KEY`)가 없으면 대기질만으로 100점 만점 계산합니다.

---

## 가중치 배분

### 기상 데이터 있을 때 (대기질 70% + 기상 30%)

| 구분 | 항목 | 가중치 |
|---|---|---|
| 대기질 | 초미세먼지 (PM2.5) | 35% |
| 대기질 | 미세먼지 (PM10) | 20% |
| 대기질 | 오존 (O₃) | 15% |
| 기상 | 기온 | 12% |
| 기상 | 습도 | 8% |
| 기상 | 풍속 | 5% |
| 기상 | 강수 | 5% |

### 기상 데이터 없을 때 (대기질 100%)

| 항목 | 가중치 |
|---|---|
| 초미세먼지 (PM2.5) | 50% |
| 미세먼지 (PM10) | 30% |
| 오존 (O₃) | 20% |

## 대기질 페널티 산출 (0~100)

### 초미세먼지 (PM2.5, μg/m³)

| 구간 | 페널티 | 계산식 |
|---|---|---|
| 0 ~ 15 | 0 | 좋음 |
| 16 ~ 35 | 0 ~ 40 | `(v - 15) / 20 × 40` |
| 36 ~ 75 | 40 ~ 80 | `40 + (v - 35) / 40 × 40` |
| 76 이상 | 100 | 최대 페널티 |

### 미세먼지 (PM10, μg/m³)

| 구간 | 페널티 | 계산식 |
|---|---|---|
| 0 ~ 30 | 0 | 좋음 |
| 31 ~ 80 | 0 ~ 40 | `(v - 30) / 50 × 40` |
| 81 ~ 150 | 40 ~ 80 | `40 + (v - 80) / 70 × 40` |
| 151 이상 | 100 | 최대 페널티 |

### 오존 (O₃, ppm)

| 구간 | 페널티 | 계산식 |
|---|---|---|
| 0 ~ 0.03 | 0 | 좋음 |
| 0.031 ~ 0.09 | 0 ~ 50 | `(v - 0.03) / 0.06 × 50` |
| 0.091 이상 | 100 | 최대 페널티 |

#### 야간 감경 (20시 ~ 06시)

`getRunningIndex`에 `hour`를 전달하면 해당 시간대의 O₃ 페널티를 **30%로 감경**합니다.

```
if (hour >= 20 || hour < 7) penalty = basePenalty × 0.3
```

- **이유:** 지표면 오존은 광화학 반응(UV + NOx + VOC)으로 생성되므로 해가 진 뒤에는 생성이 멈추고,
  도심에선 NO 적정(NO + O₃ → NO₂)으로 잔여 O₃가 빠르게 소모됩니다. 실제 야간 농도가 낮 대비 매우
  낮은 상황에서 고정 임계값(0.03 ppm~)을 그대로 적용하면 센서 잔류값이나 교외 배경 농도에
  과도하게 감점되는 문제가 있어 가중치를 낮춥니다.
- `hour`를 전달하지 않으면(=`undefined`) 기존 동작과 동일하게 전 시간대 동일 페널티를 사용합니다.

## 기상 페널티 산출 (0~100)

### 기온 (°C) — 최적: 12~22°C

| 구간 | 페널티 | 설명 |
|---|---|---|
| 12 ~ 22 | 0 | 러닝 최적 온도 |
| 5 ~ 11 | 0 ~ 30 | 약간 쌀쌀함 |
| 23 ~ 28 | 0 ~ 30 | 약간 더움 |
| 0 ~ 4 | 30 ~ 70 | 추움 |
| 29 ~ 35 | 30 ~ 70 | 더움 |
| 0°C 미만 / 35°C 초과 | 100 | 야외 운동 부적합 |

### 습도 (%) — 최적: 40~60%

| 구간 | 페널티 | 설명 |
|---|---|---|
| 40 ~ 60 | 0 | 쾌적 |
| 30 ~ 39 | 0 ~ 20 | 약간 건조 |
| 61 ~ 80 | 0 ~ 30 | 약간 습함 |
| 81 ~ 100 | 30 ~ 100 | 고습 — 체온 조절 어려움 |
| 30 미만 | 30 | 건조 |

### 풍속 (m/s) — 최적: 0~3m/s

| 구간 | 페널티 | 설명 |
|---|---|---|
| 0 ~ 3 | 0 | 미풍, 쾌적 |
| 3 ~ 7 | 0 ~ 20 | 약풍, 역풍 시 약간 불편 |
| 7 ~ 10 | 20 ~ 50 | 강풍, 러닝 어려움 |
| 10 ~ 15 | 50 ~ 100 | 돌풍, 위험 |

### 강수 형태

가중치 기반 페널티와 **강제 감점**이 동시에 적용됩니다.

| 형태 | 가중치 페널티 | 강제 감점 | 실질 최대 감점 | 설명 |
|---|---|---|---|---|
| 없음 | 0 | 0 | 0 | — |
| 비 | 80 (×5% = 4) | **30** | **34** | 노면 미끄러움, 시야 저하 |
| 진눈깨비 | 90 (×5% = 4.5) | **45** | **49.5** | 비 + 눈 혼합 |
| 눈 | 100 (×5% = 5) | **50** | **55** | 야외 러닝 부적합 |

> 강제 감점은 가중치 곱셈과 별개로 점수에서 직접 차감합니다.
> 비가 오면 다른 조건이 아무리 좋아도 "자제" 이하, 눈이면 거의 확실히 "금지"로 내려갑니다.

## 복합 악화 보정

가중치 곱셈으로 인해 개별 항목의 감점이 희석되는 문제를 보완합니다.
**개별 페널티 > 25인 항목이 2개 이상** 동시에 나쁘면, 가중 합산에 추가 감점을 적용합니다.

### 계산식

```
significantPenalties = rawPenalties.filter(p > 25)
count               = significantPenalties.length

count < 2이면 → 추가 감점 없음

avgSeverity = sum(significantPenalties) / (count × 100)
compound    = count × avgSeverity × 20

최종 페널티 = 기본 가중 페널티 + compound
```

### 예시

| 항목 | 개별 페널티 | > 25? |
|---|---|---|
| PM2.5 (40 μg/m³) | 45 | O |
| PM10 (55 μg/m³) | 20 | X |
| O₃ (0.05 ppm) | 17 | X |
| 기온 (18°C) | 0 | X |
| 습도 (85%) | 48 | O |
| 강수 (없음) | 0 | X |

```
기본 가중 페널티 = 45×0.35 + 20×0.20 + 17×0.15 + 0×0.15 + 48×0.10 + 0×0.05 = 27.0
significant = [45, 48] → count = 2
avgSeverity = (45 + 48) / 200 = 0.465
compound    = 2 × 0.465 × 20 = 18.6
최종 페널티 = 27.0 + 18.6 = 45.6
점수        = 100 - 46 = 54 → '주의 필요'
```

보정 없이는 점수 73("달리기 좋음")이 되어 체감과 맞지 않는 문제를 해결합니다.

## 점수 → 상태 매핑

| 점수 | 상태 | 라벨 | 의미 |
|---|---|---|---|
| 80 ~ 100 | `great` | 달리기 최적 | 지금 바로 달리세요! |
| 60 ~ 79 | `good` | 달리기 좋음 | 좋은 환경입니다 |
| 40 ~ 59 | `caution` | 주의 필요 | 짧은 러닝 권장 |
| 20 ~ 39 | `bad` | 달리기 자제 | 실내 운동 권장 |
| 0 ~ 19 | `worst` | 달리기 금지 | 야외 활동 자제 |

`canRun` 플래그는 점수 40 이상일 때 `true`입니다.

각 오염물질의 현재 수준을 사용자에게 표시할 때 아래 6단계로 분류합니다.
러닝 지수 점수 계산과는 별개이며, 대기질 상세 패널에서 사용됩니다.

### 초미세먼지 (PM2.5, μg/m³)

| 등급 | 범위 | 색상 | 설명 |
|---|---|---|---|

| 보통 | 16 ~ 25 | 파랑 | 대부분의 사람에게 무리 없는 수준입니다. |
| 민감군주의 | 26 ~ 35 | 노랑 | 민감군은 장시간 격렬한 운동을 줄이세요. |

| 매우나쁨 | 76 이상 | 빨강 | 모든 사람이 실외 활동을 자제해야 합니다. |

## 24시간 타임라인 예보

### 데이터 소스 우선순위

1. **실측값** — 에어코리아 API에서 받은 시간별 측정값
2. **과거 빈 데이터** — 현재 측정값으로 대체
3. **미래 시간** — 현재 측정값에 시간대 패턴 적용

### 시간대 패턴 (미래 예측용)

| 시간대 | 배수 | 이유 |
|---|---|---|
| 출퇴근 (7~9시, 17~19시) | ×1.4 | 차량 매연 급증 |
| 새벽 (3~6시) | ×0.6 | 교통량 감소 |
| 그 외 | ×1.0 | 기본 |

### 최적 시간 선정

24시간 예보 중 **점수 ≥ 60 (달리기 좋음 이상)** 시간대를 추려 점수 내림차순 상위 3개를 추천합니다.
